\section{Results}

\begin{frame}{Results \& Demo}
  \begin{itemize}
    \item Built deliverables: trained benchmark suite, IEEE report, Streamlit dashboard, and presentation deck.
    \item Main benchmark: C-MAPSS FD001 comparing baseline, fixed loop, adaptive loop, and LLM baseline.
    \item Main quantitative claim: the adaptive loop beat the LLM baseline on both macro F1 and measured latency.
    \item Live demo artifact: dashboard sample bundle with per-window predictions, maintenance actions, subsystem attribution, and adaptive depth traces.
  \end{itemize}
\end{frame}

\begin{frame}{C-MAPSS FD001 Test Results}
  \begin{tabular}{lcccc}

\vspace{0.4cm}
  \begin{itemize}
    \item The main project comparison is adaptive loop versus the LLM baseline.
    \item Best macro F1 came from the adaptive loop.
    \item Average loop compute dropped by about 79.8\% versus fixed depth.
    \item The adaptive model was about 101x faster per sample than the LLM baseline.
    \item Critical-class performance remained strong with perfect precision and 0.75 recall on the small FD001 critical subset.
  \end{itemize}
\end{frame}

\begin{frame}{Result Interpretation}
  \begin{itemize}
    \item Fixed depth won on validation macro F1, but adaptive depth generalized better on test.
    \item Adaptive looping improved class balance, especially beyond the dominant normal class.
    \item The LLM baseline kept decent nominal accuracy but collapsed on macro F1 because it under-served minority failure states.
    \item The main value is not just accuracy; it is better triage under a lower average compute budget than both recurrent and LLM alternatives.
    \item Most remaining errors were normal-versus-warning confusions rather than full failure-state collapse.
  \end{itemize}
\end{frame}
