\section{Limitations}

\begin{frame}{Broader Impact \& Limitations}
  \begin{itemize}
    \item A task-specific looped model can be a better fit than a generic text-first LLM pipeline for numeric maintenance telemetry.
    \item CMAPSS is simulated telemetry, not a live maintenance deployment stream.
    \item IMS labels are still derived rather than ground-truth maintenance annotations.
    \item Action and subsystem targets are coarse supervisory proxies, not technician-authored labels.
    \item Latency is now benchmarked explicitly, but current numbers are local-device measurements rather than deployment SLAs.
  \end{itemize}
\end{frame}
