\section{Motivation}

\begin{frame}{Problem Statement}
  \begin{itemize}
    \item Reduce unplanned downtime by detecting degradation early.
    \item Prioritize maintenance by urgency instead of reacting to failures.
    \item Use adaptive compute so easy windows exit early and ambiguous windows get more reasoning.
    \item Align the model with the project hypothesis: looped latent refinement should help on noisy sequential telemetry.
  \end{itemize}
\end{frame}

\begin{frame}{Why This Matters}
  \begin{itemize}
    \item Input: rolling telemetry windows plus maintenance context.
    \item Output: \texttt{normal}, \texttt{warning}, or \texttt{critical}.
    \item Secondary output: suggested maintenance action.
    \item Core question: can a looped model beat an LLM baseline on maintenance triage at lower average compute?
  \end{itemize}
\end{frame}
