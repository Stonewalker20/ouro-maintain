\section{Motivation}

\begin{frame}{Problem Statement}
  \begin{itemize}
    \item Machines often show warning signs before they fail.
    \item If we catch those signs early, we can fix equipment before it causes downtime.
    \item The challenge is that sensor data is noisy, and not every case is equally hard.
    \item We wanted a model that thinks a little on easy cases and longer on harder ones.
  \end{itemize}
\end{frame}

\begin{frame}{What We Wanted the System To Do}
  \begin{itemize}
    \item Read a short window of sensor data.
    \item Decide whether the machine looks \texttt{normal}, \texttt{warning}, or \texttt{critical}.
    \item Suggest what a technician should do next.
    \item Test whether this task-specific looped model works better than a task-specific LLM.
  \end{itemize}
\end{frame}
