\section{Experiments}
\subsection{Experimental Protocol}
All reported primary results use C-MAPSS FD001. Training uses the official run-to-failure trajectories, validation uses an engine-level held-out split from the training set, and testing uses the official FD001 test split with its published RUL labels. Each model is trained for six epochs with batch size 128, hidden size 128, and a maximum loop depth of six steps. The adaptive model uses the same backbone as the fixed-depth loop and differs only in the early-exit policy. The current codebase also supports multitask supervision for maintenance action prediction and coarse subsystem attribution, along with stage-aware temporal validation for IMS.

\subsection{Metrics}
Classification results are reported with accuracy, macro F1, and weighted F1. Maintenance-specific reporting emphasizes critical-failure recall, action accuracy, subsystem accuracy, average loop depth, and measured latency in milliseconds per sample. Loop depth remains useful as an internal compute proxy, but the implementation now writes explicit latency benchmarks for validation and test artifacts as well.

\subsection{Ablations}
This report focuses on the core comparison requested by the project concept: direct prediction, fixed-depth looping, and adaptive looping under a shared maximum depth. Useful follow-on ablations now include the multitask loss weights, the stage-aware IMS label boundaries, maximum loop depth, and the exit threshold.
