\section{Results}
\subsection{Results and Demo}
The primary quantitative results come from C-MAPSS FD001. The adaptive looped model produced the strongest macro F1 score on the held-out test set while using far fewer internal steps than the fixed-depth loop and far lower latency than both the frozen and task-adapted LLM baselines.

\subsection{Primary Findings}
\begin{itemize}
    \item The adaptive model achieved test macro F1 of 0.9183, outperforming the fixed-depth loop at 0.8891, the direct baseline at 0.8609, the frozen LLM baseline at 0.3183, and the task-adapted LLM baseline at 0.3454.
    \item The adaptive model required an average of only 1.21 recurrent steps on the test set, compared with 6.0 steps for the fixed-depth loop.
    \item Relative to the fixed-depth loop, the adaptive policy reduced average loop computation by approximately 79.8\%.
    \item The adaptive model averaged 0.17 ms/sample on CPU latency benchmarking, compared with 16.95 ms/sample for the LLM baseline, or roughly a 101x latency advantage.
    \item A stronger task-adapted DistilBERT baseline improved the LLM score to 0.3454 macro F1, but it still remained 0.5729 behind the adaptive loop and raised latency to 32.11 ms/sample.
    \item The adaptive model preserved strong critical-class performance, reaching perfect precision and 0.75 recall on the small critical test subset.
    \item Across the full local dataset matrix, the adaptive model outperformed both the frozen and task-adapted LLM baselines on every CMAPSS subset and all three IMS validation runs.
    \item On the reduced Paderborn machinery benchmark, the adaptive model reached validation macro F1 of 0.2811 versus 0.1838 for both frozen and task-adapted LLM baselines.
    \item On the reduced HVAC extension split, the adaptive model reached validation macro F1 of 0.6000 with 1.55 average loop steps and 1.50 ms/sample latency.
\end{itemize}

\subsection{Validation and Generalization}
On validation, the fixed-depth loop reached the best macro F1 score of 0.9366, while the adaptive model reached 0.9142 at an average depth of 1.39 steps. On the official test split, however, the adaptive model generalized better and delivered the strongest macro F1 score. The frozen LLM baseline, despite strong nominal accuracy from the dominant normal class, collapsed on macro F1 because it failed to separate warning and critical states reliably. The stronger task-adapted LLM narrowed that gap only slightly, rising from 0.3183 to 0.3454 macro F1 on the FD001 test set. This is the central comparison of the project: the looped numeric model handled minority degradation states far better than both generic and task-adapted text-transformer baselines while also avoiding the heavy latency cost of tokenization and transformer inference. The pattern suggests that fixed full-depth recurrence may fit the development split more aggressively, while adaptive early exit regularizes the model enough to improve test-time class balance without paying the cost of text-serialized inference.

\subsection{Secondary Dataset Status}
The IMS pipeline uses extracted run-to-failure vibration snapshots, summary-statistic feature generation, and cached preprocessing. Early temporal-split runs on IMS showed degenerate validation folds with missing classes, so the reported IMS results use validation-only evaluation. Under that setup, the adaptive model remained strong across the first, second, and fourth IMS runs, while both frozen and task-adapted LLM baselines stayed clustered near macro F1 values of 0.24--0.25. On the C-MAPSS subsets, task adaptation did improve DistilBERT materially over the frozen baseline, but the adaptive loop still retained a large margin on every official test split. The main quantitative claims remain anchored on C-MAPSS because it provides official train/test splits, but the full matrix shows that the adaptive-versus-task-adapted-LLM gap is not limited to a single FD001 slice.

\subsection{Cross-Domain Extension Results}
Table~\ref{tab:extension-results} summarizes the additional domain checks added after the main C-MAPSS study. The Paderborn benchmark is the completed new-domain loop-versus-LLM comparison because all three model families finished under the same reduced machinery split. There, the adaptive loop improved macro F1 by 0.0973 over both LLM baselines while remaining about 556x faster than the frozen LLM and about 308x faster than the task-adapted LLM on sample latency.

The HVAC extension is reported as a looped-model transfer study rather than an LLM comparison. On the reduced asset-stratified split, the adaptive model reached validation macro F1 of 0.6000 and achieved perfect recall on the critical class. Because the matching HVAC LLM runs were not practical on the available laptop CPU, they are omitted from the final benchmark matrix rather than reported as incomplete entries.

\subsection{Demo Deliverable}
The completed system includes a Streamlit dashboard for live presentation use. The dashboard loads saved artifacts, displays benchmark comparisons, renders confusion matrices and training curves, and lets a user inspect individual telemetry windows with predicted health state, maintenance action, subsystem attribution, confidence, latency, and adaptive exit depth. That demo makes the quantitative comparison tangible by showing how the model behaves on specific examples instead of only aggregate tables.
