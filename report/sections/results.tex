\section{Results}
\subsection{Results and Demo}
The primary quantitative results come from C-MAPSS FD001. The adaptive looped model produced the strongest macro F1 score on the held-out test set while using far fewer internal steps than the fixed-depth loop and far lower latency than the text-serialized LLM baseline.

\subsection{Primary Findings}
\begin{itemize}
    \item The adaptive model achieved test macro F1 of 0.9183, outperforming the fixed-depth loop at 0.8891, the direct baseline at 0.8609, and the LLM baseline at 0.3183.
    \item The adaptive model required an average of only 1.21 recurrent steps on the test set, compared with 6.0 steps for the fixed-depth loop.
    \item Relative to the fixed-depth loop, the adaptive policy reduced average loop computation by approximately 79.8\%.
    \item The adaptive model averaged 0.17 ms/sample on CPU latency benchmarking, compared with 16.95 ms/sample for the LLM baseline, or roughly a 101x latency advantage.
    \item The adaptive model preserved strong critical-class performance, reaching perfect precision and 0.75 recall on the small critical test subset.
\end{itemize}

\subsection{Validation and Generalization}
On validation, the fixed-depth loop reached the best macro F1 score of 0.9366, while the adaptive model reached 0.9142 at an average depth of 1.39 steps. On the official test split, however, the adaptive model generalized better and delivered the strongest macro F1 score. The LLM baseline, despite strong nominal accuracy from the dominant normal class, collapsed on macro F1 because it failed to separate warning and critical states reliably. This is the central comparison of the project: the looped numeric model handled minority degradation states far better than the pretrained text-transformer baseline while also avoiding the heavy latency cost of tokenization and transformer inference. The pattern suggests that fixed full-depth recurrence may fit the development split more aggressively, while adaptive early exit regularizes the model enough to improve test-time class balance without paying the cost of text-serialized inference.

\subsection{Secondary Dataset Status}
The IMS pipeline uses extracted run-to-failure vibration snapshots, summary-statistic feature generation, and cached preprocessing. Early temporal-split runs on IMS showed degenerate validation folds with missing classes, so the reported IMS results use stratified validation instead. Under that evaluation setup, the adaptive model performed strongly across the first, second, and fourth IMS runs. The main quantitative claims remain anchored on C-MAPSS because it provides official train/test splits, while IMS serves as a complementary secondary benchmark.

\subsection{Demo Deliverable}
The completed system includes a Streamlit dashboard for live presentation use. The dashboard loads saved artifacts, displays benchmark comparisons, renders confusion matrices and training curves, and lets a user inspect individual telemetry windows with predicted health state, maintenance action, subsystem attribution, confidence, latency, and adaptive exit depth. That demo makes the quantitative comparison tangible by showing how the model behaves on specific examples instead of only aggregate tables.
