\section{Discussion}
The results are most meaningful when interpreted in practical maintenance terms rather than only as aggregate ML metrics. In that framing, the strongest outcome is that the adaptive looped model preserves useful recall on critical failures while reducing average internal depth on routine windows. This behavior supports the claim that iterative latent reasoning is useful for maintenance triage, where most cases are straightforward but a small subset requires deeper inspection.

\subsection{Threats to Validity}
The main threats are dataset bias, label construction choices, and split leakage. C-MAPSS thresholds can materially change class balance, while IMS labels are derived rather than observed directly. Cross-dataset comparisons therefore provide evidence of generalization behavior rather than a perfect apples-to-apples comparison of supervisory signal.

\subsection{Interpretation Checklist}
\begin{itemize}
    \item Check whether hard cases consistently consume more loop steps than easy cases.
    \item Check whether depth savings come from true early exits rather than overconfident misclassifications.
    \item Check whether the adaptive model stays calibrated under class imbalance.
    \item Check whether C-MAPSS and IMS agree on the same qualitative degradation patterns.
\end{itemize}
